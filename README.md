# Прототип: детекция событий у столика по видео

Упрощённый пайплайн для одного ролика: детекция людей (YOLOv8n), одна зона столика (ручное выделение), события «пусто / занято / подход после пустоты», таблица событий в Pandas, метрика интервалов «пусто → следующий подход» и выходное видео с подсветкой зоны.

## Требования

- **Python 3.11+** (в коде используются `StrEnum`, `typing.Self`).
- Видеофайл в формате, который открывает OpenCV (например MP4).

## Установка

```bash
python -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

При первом запуске Ultralytics подтянет веса `yolov8n.pt` (в текущую рабочую директорию или кэш пакета).

## Запуск

```bash
python main.py --video путь/к/video1.mp4
```

### Куда пишутся файлы

Пути из аргументов **относительные** склеиваются с каталогами из `DetectionConfig` (корень репозитория = родитель `src/`):

| CLI (по умолчанию) | Каталог | Итоговый путь (пример) |
|--------------------|---------|-------------------------|
| `--output` → `output.mp4` | `output_videos/` | `<корень>/output_videos/output.mp4` |
| `--report` → `report.txt` | `reports/` | `<корень>/reports/report.txt` |
| `--problem-frame` → `problem_frame.png` | `problem_frames/` | `<корень>/problem_frames/problem_frame.png` |

**Абсолютный** путь в `--output` / `--report` / `--problem-frame` используется как есть (без префикса этих каталогов). Родительские каталоги создаются при старте пайплайна.

### Опции CLI

| Параметр | По умолчанию | Назначение |
|----------|---------------|------------|
| `--video` | — | Входное видео (обязательно) |
| `--output` | `output.mp4` | Имя/путь выходного видео с визуализацией |
| `--report` | `report.txt` | Текстовый отчёт и CSV событий |
| `--problem-frame` | `problem_frame.png` | Кадр-иллюстрация («проблемный» момент) |
| `--problem-at` | `0.35` | Доля длины ролика (0…1), на какой позиции сохранить кадр |
| `-v` / `--verbose` | выкл. | Уровень логов DEBUG |
| `--no-progress` | выкл. | Без прогресс-бара кадров (CI / перенаправленный stderr) |

Пример с явными путями (относительные имена попадут в каталоги выше):

```bash
python main.py --video ./media/cafe.mp4 --output run1.mp4 --report run1.txt --problem-frame run1.png
```

После старта откроется окно **с первым кадром**: мышью выделите прямоугольник зоны **одного столика** и нажмите **Enter** или **Space** (`cv2.selectROI`). Пустой ROI (ширина или высота 0) завершит программу с сообщением об ошибке.

### Прерывание (Ctrl+C)

Обработчик в `main.py`: короткое сообщение в лог, код выхода **130**. Ресурсы OpenCV освобождаются в `TableVideoPipeline.__exit__`. **Частично записанные** выходные файлы при Ctrl+C **удаляются**.

## Какое видео и какой столик

- **Видео:** задаётся в `--video` (демо-ролик в репозитории не обязателен).
- **Столик:** координаты зоны **не зашиты** — задаются интерактивно при каждом запуске. В отчёте сохраняются `(x, y, width, height)` выбранного ROI.

## Логика детекции событий

1. **Люди:** YOLOv8n (`ultralytics`), класс person (`DetectionConfig.person_class_id`). Порог уверенности — `yolo_conf` (по умолчанию `0.35`). Для каждого бокса считается доля площади пересечения с зоной стола; если она **не ниже** `person_table_overlap_ratio` (по умолчанию `0.15`), человек считается **в зоне**.
2. **Сглаживание:** конечный автомат с дебаунсом по кадрам (`debounce_frames`, по умолчанию `10`).
3. **События** (в таблице колонка `event`):
   - `empty` — логически стол пуст;
   - `occupied` — стол занят;
   - `approach` — эпизод «подход» после периода пустоты (логика в `DebouncedTableStateMachine`).

События попадают в **Pandas DataFrame** (`frame`, `time_sec`, `event`), в `report.txt` дублируются как CSV-блок.

## Результат работы

- **Видео** — то же разрешение, что у входа: прямоугольник зоны стола **зелёный** (пусто) / **красный** (занято), боксы людей — **синие** (BGR в OpenCV).
- **`report.txt`** — заголовок, путь к видео, зона стола, параметры детекции, CSV событий, блок **Metrics: empty table -> next approach** (список интервалов в секундах и среднее или `N/A`).

Средняя задержка зависит от конкретного ролика — смотрите консоль (`EventAnalytics.log_summary`) и конец отчёта.

## Пример кадра

По умолчанию сохраняется кадр около доли `--problem-at` в `problem_frames/problem_frame.png` (или по вашему `--problem-frame`).

## Структура проекта

- `main.py` — точка входа, логирование, обработка `KeyboardInterrupt`.
- `src/cli.py` — разбор аргументов.
- `src/configs.py` — `DetectionConfig`, каталоги артефактов.
- `src/logging_config.py` — настройка `logging`.
- `src/pipeline.py` — видео, ROI, цикл кадров, writer, отчёт.
- `src/person_detector.py` — YOLO и пересечение с `TableZone`.
- `src/debounced_table_fsm.py` — события состояния стола.
- `src/domain.py` — `TableZone`, `BoundingBox`, `TableEvent`, перечисление видов событий.
- `src/analytics.py` — DataFrame и метрики «пусто → подход».
- `src/visualization.py` — ROI, отрисовка зоны и людей.
