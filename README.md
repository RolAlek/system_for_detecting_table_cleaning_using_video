# Прототип: детекция событий у столика по видео

Упрощённый пайплайн для одного ролика: детекция людей (YOLOv8n), одна зона столика (ручное выделение), события «пусто / занято / подход после пустоты», таблица событий в Pandas, базовая метрика задержки и выходное видео с подсветкой зоны.

## Требования

- **Python 3.11+** (в коде используются `StrEnum`, `typing.Self`).
- Видеофайл в формате, который открывает OpenCV (например MP4).

## Установка

```bash
python -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

При первом запуске Ultralytics подтянет веса `yolov8n.pt`.

## Запуск

```bash
python main.py --video путь/к/video1.mp4
```

Полезные опции:

| Параметр | По умолчанию | Назначение |
|----------|---------------|------------|
| `--output` | `output.mp4` | Видео с визуализацией |
| `--report` | `report.txt` | Текстовый отчёт и CSV событий |
| `--problem-frame` | `problem_frame.png` | Сохранённый кадр (иллюстрация / «проблемный» момент) |
| `--problem-at` | `0.35` | Доля длины ролика (0…1), на какой позиции сохранить кадр |

Пример:

```bash
python main.py --video ./data/cafe.mp4 --output ./out/output.mp4 --report ./out/report.txt
```

После старта откроется окно **с первым кадром**: мышью выделите прямоугольник зоны **одного столика** и нажмите Enter или Space (`cv2.selectROI`).

## Какое видео и какой столик

- **Видео:** указываете вы сами в `--video` (в репозитории демо-ролик не хранится).
- **Столик:** координаты зоны **не зашиты в код** — задаются интерактивно при каждом запуске. В отчёте `report.txt` сохраняются `(x, y, ширина, высота)` выбранного ROI.

Для отчёта по заданию: после прогона впишите в этот README фактическое имя файла и при необходимости вставьте скрин (например `problem_frame.png`) в репозиторий и дайте ссылку на файл в разделе ниже.

## Логика детекции событий

1. **Люди:** YOLOv8n (`ultralytics`), класс «person». Для каждого бокса считается доля площади пересечения с зоной стола; если она не ниже порога (`DetectionConfig.person_table_overlap_ratio`, по умолчанию `0.15`), считается, что человек **в зоне**.
2. **Сглаживание:** конечный автомат с дебаунсом по кадрам (`debounce_frames`, по умолчанию `10`), чтобы убрать дрожание детектора.
3. **События:**
   - `empty` — логически стол пуст (никого в зоне после стабилизации);
   - `occupied` — стол занят;
   - `approach` — переход к «занят» после периода «пусто» (фиксируется вместе с `occupied` на кадре смены состояния).

События пишутся в **Pandas DataFrame** (колонки: `frame`, `time_sec`, `event`), дублируются в `report.txt`.

## Результат работы

- **`output.mp4`** — то же разрешение, что у входа: прямоугольник зоны стола **зелёный** (пусто) / **красный** (занято), боксы людей — оранжевые.
- **`report.txt`** — зона стола, параметры детекции, таблица событий, строка со **средним временем между событием «стол пуст» и следующим «подход»** (в секундах), если такие пары есть.

**Средняя задержка для конкретного видео** не фиксируется в репозитории — она получается только после вашего прогона; смотрите консоль и конец `report.txt`.

## Пример кадра

После запуска по умолчанию сохраняется **`problem_frame.png`** (кадр около `--problem-at` от длины ролика). Его можно приложить к отчёту как иллюстрацию сцены или спорного момента детекции.

## Структура проекта

- `main.py` — точка входа.
- `src/pipeline.py` — чтение видео, ROI, цикл кадров, запись результата.
- `src/person_detector.py` — YOLO и проверка пересечения с `TableZone`.
- `src/debounced_table_fsm.py` — события состояния стола.
- `src/analytics.py` — DataFrame и метрика «пусто → следующий подход».
- `src/visualization.py` — ROI, отрисовка зоны и людей.
